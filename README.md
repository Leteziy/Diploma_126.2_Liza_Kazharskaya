# Diploma_126.2_Liza_Kazharskaya
 Pепозиторий на GitHub с автоматизацией тест-кейсов проекта "Читай-город" 
Документация проекта
Проект использует следующие технологии и синтаксис:

Python: Основной язык программирования для написания тестов.
Selenium: Библиотека для автоматизации взаимодействия с веб-браузером.
Pytest: Фреймворк для написания и запуска тестов.
Requests: Библиотека для отправки HTTP-запросов и тестирования API-эндпоинтов.
Allure: Инструмент для генерации отчетов о выполнении тестов.
Форматирование кода.
Python-dotenv: Инструмент для автоматической загрузки конфигурационных данных и секретов из файлов переменных окружения.

Код форматируется в соответствии с PEP 8 (стиль написания кода на Python).
Используются docstrings для документирования методов и функций.

В проекте присутствуют следующие файлы: 
 .env — Файл переменных окружения для безопасного хранения конфиденциальных данных и настроек (URL, токенов) отдельно от исходного кода. .env.example - файл дублирует содержание .env, но без секретных данных, передается в ссылке на ознакомление работы.
 config.py — Централизованный конфигурационный файл, который считывает данные из .env и экспортирует их в автотесты в виде готовых переменных.
 pytest.ini — Основной конфигурационный файл фреймворка PyTest, содержащий настройки путей поиска модулей, правила запуска и маркеры тестов.
 .gitignore - предотвращает случайное добавление в общий репозиторий конфиденциальных данных, временных документов и результатов сборки.
 conftest.py - содержит общие фикстуры проекта, в том числе настройки браузера (driver).

Основная папка тестов называется test. В ней содержатся подпапки ui, api. В подпапке ui  находится 5 тестов, использующих PageObject-классы. Подпапка api содержит 6 api-тестов.

Проект поддерживает три режима запуска через маркеры:
-Запуск только UI-тестов pytest -m ui,
-Запуск только API-тестов pytest -m api,
-Запуск всех тестов pytest.

Все тесты содержат декораторы @allure.title, allure.severity, @allure.feature, @allure.story.
Все шаги теста размечаются с помощью @allure.step или with allure.step для улучшения читаемости отчетов.

Инструкция по запуску тестов для формирования отчета Allure
Установите необходимые зависимости:

pip install -r requirements.txt

Запустите тесты с генерацией отчета Allure:

pytest --alluredir=./allure-results

Эта команда запустит все тесты и сохранит результаты в директорию ./allure-results.

Инструкция по просмотру сформированного отчета Allure
После завершения тестов, чтобы просмотреть отчет, выполните команду:

allure serve ./allure-results

Эта команда запустит локальный сервер и откроет отчет в браузере.

В отчете Allure вы увидите: Выполненные тесты в разделе Suites.
При клике на тесты в этом разделе появятся: Название теста, описание теста, серьезность, время выполнения теста, шаги теста.
